## Overview

This repository contains a carefully curated learning path designed to transform a software engineer into a well-rounded software architect. The curriculum spans 2.5 years and covers 30 foundational texts in software engineering, architecture, and design.

### Vision

To build a comprehensive understanding of software architecture through progressive learning phases, starting from code-level practices and advancing to system-level architectural thinking.

### Target Audience

Software engineers with 2-5 years of experience seeking to:
- Transition into software architecture roles
- Deepen their understanding of software design principles
- Master enterprise-level architecture patterns
- Build a solid foundation in software engineering theory and practice

## Learning Best Practices

### Weekly Rhythm
- **Monday-Friday:** 1-2 hours daily reading
- **Saturday:** Review and note-taking session
- **Sunday:** Practical application in personal projects

### Monthly Milestones
1. **Week 1:** Read first 25% + initial notes
2. **Week 2:** Read next 25% + concept mapping
3. **Week 3:** Read next 25% + practical exercises
4. **Week 4:** Complete reading + summary & review

### Practical Application Strategy
- **Immediate Application:** Apply concepts in daily work
- **Side Projects:** Create reference implementations
- **Code Reviews:** Share knowledge with team
- **Teaching:** Blog posts or internal presentations

### Knowledge Retention Techniques
- **Active Reading:** Annotate and question
- **Spaced Repetition:** Review notes monthly
- **Concept Mapping:** Visual knowledge graphs
- **Peer Discussion:** Study groups or book clubs
- **Writing:** Technical blog posts

## Assessment & Milestones

### Self-Assessment Checklist

#### After Year 1
- [ ] Can explain and apply all major design patterns
- [ ] Write tests before code naturally

- [ ] Design clean, SOLID-compliant interfaces
- [ ] Mentor others on code quality

#### After Year 2
- [ ] Design complete system architectures
- [ ] Document architectural decisions clearly
- [ ] Evaluate architectural trade-offs
- [ ] Understand distributed system challenges
- [ ] Lead architectural discussions

#### After 2.5 Years
- [ ] Think strategically about technology choices
- [ ] Balance technical and business concerns
- [ ] Create technical roadmaps
- [ ] Guide multiple teams architecturally
- [ ] Contribute to engineering culture
